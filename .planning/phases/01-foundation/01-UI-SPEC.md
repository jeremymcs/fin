# Phase 1: Foundation — UI Design Contract

> Visual and interaction contract for the TUI rendering layer. Generated by gsd-ui-researcher.
> This is a terminal UI (ratatui 0.29 / crossterm 0.28), not a web frontend.
> All design tokens map to ratatui `Color`, `Modifier`, and `Style` primitives — not CSS.
> shadcn gate: not applicable (Rust/Ratatui project, no React/Next.js/Vite).

## Spacing Scale

Terminal UIs do not use pixel spacing. Spacing is expressed in character cells and layout constraints.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 cell | Inline span separator (e.g. `  ·  ` between extension names) |
| sm | 2 cells | Indent prefix for bullet/numbered list items (`  • `, `  N `) |
| md | 4 cells | Workflow panel annotation indent (`  ↳ `) |
| lg | Constraint::Length(1) | Spacer row between output and status bar |
| xl | Constraint::Length(4) | Workflow panel height (default) |
| 2xl | Constraint::Length(1) | Status bar height |
| input | Constraint::Length(3) | Input box height |

Exceptions: none. Layout constraints are defined in `AppLayout::compute()` — not scattered inline.

Source: RESEARCH.md Pattern 2 (AppLayout struct), CONTEXT.md D-05/D-06

## Typography (Terminal Modifier Contract)

Terminal UI has no font sizes. Typography maps to ratatui `Modifier` flags and `Color` values.

| Role | Modifier | Color | Usage |
|------|----------|-------|-------|
| Body text | none | `Palette::TEXT` (White) | Assistant plain text, system messages |
| Bold emphasis | `Modifier::BOLD` | `Palette::TEXT` (White) | Markdown `**bold**` spans in assistant output |
| Italic emphasis | `Modifier::ITALIC` | `Palette::TEXT` (White) | Markdown `*italic*` spans in assistant output |
| Inline code | `Modifier::REVERSED` | terminal default | Markdown `` `code` `` spans — helix convention; fallback `BOLD \| DIM` if reversed causes theme issues |
| Dim / subdued | `Modifier::DIM` | `Palette::DIM` (DarkGray) | Thinking lines, separators, per-message cost annotation |
| Accent label | `Modifier::BOLD` | `Palette::ACCENT` (Yellow) | Splash labels, input prompt, workflow border title, active stage marker |
| User text | `Modifier::BOLD` | `Palette::SUCCESS` (Green) | User message lines |
| Error text | `Modifier::BOLD` | `Palette::ERROR` (Red) | Error line kinds |

Line height: single-line cells. Ratatui wraps at widget boundary — no additional configuration.

Source: CONTEXT.md D-07/D-08/D-09/D-10, RESEARCH.md Pattern 1

## Color Contract (Palette Struct)

All colors must be declared as associated constants on a `struct Palette` in `widgets.rs`. No inline `Color::` literals in render functions after Phase 1.

ANSI named colors only. No `Color::Rgb` or `Color::Indexed`. Source: CONTEXT.md D-04.

| Role | ANSI Value | Palette Constant | Usage (60/30/10 mapping) |
|------|-----------|-----------------|--------------------------|
| Dominant (60%) | `Color::White` | `Palette::TEXT` | All body text, content spans, plain assistant output |
| Secondary (30%) | `Color::DarkGray` | `Palette::DIM` | Borders, separators, thinking lines, subdued annotations, status bar background |
| Accent (10%) | `Color::Yellow` | `Palette::ACCENT` | Splash labels, input prompt `[model] >`, workflow panel border title, active stage `●`, bullet/numbered list prefixes `•`/`N.`, header lines `# ...`, model picker border |
| Tool highlight | `Color::Cyan` | `Palette::TOOL` | `LineKind::Tool` events (semantic swap: was Yellow, now Cyan) |
| Success | `Color::Green` | `Palette::SUCCESS` | User text, tool results (`LineKind::ToolResult`), workflow done markers `✓` |
| Destructive | `Color::Red` | `Palette::ERROR` | `LineKind::Error` lines only |
| Status background | `Color::DarkGray` | `Palette::STATUS_BG` | Status bar `bg()` — same value as DIM, separate constant for semantic clarity |

**Accent reserved for:** splash labels, input prompt prefix, workflow panel border title, active stage pipeline marker `●`, list item prefixes, markdown header lines. Never used for plain body text or success/error states.

**Color role swap (D-01/D-02):**

| Widget / LineKind | Before Phase 1 | After Phase 1 |
|-------------------|---------------|---------------|
| Splash labels ("Model", "Provider") | `Color::Cyan` bold | `Palette::ACCENT` (Yellow) bold |
| Input prompt `[model] >` | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| Workflow panel border title | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| Active stage `●` | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| Bullet prefix `•` | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| Numbered list prefix `N.` | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| Header lines `# ...` | `Color::Cyan` bold | `Palette::ACCENT` (Yellow) bold |
| Question lines (ends `?`) | `Color::Cyan` | remove heuristic or use `Palette::ACCENT` |
| Model picker border | `Color::Cyan` | `Palette::ACCENT` (Yellow) |
| `LineKind::Tool` events | `Color::Yellow` dim | `Palette::TOOL` (Cyan) dim |

Source: CONTEXT.md D-01/D-02/D-03/D-04, RESEARCH.md Pattern 1 color role table

## Copywriting Contract

Terminal output strings used as UI copy in the rendered widget layer.

| Element | Copy |
|---------|------|
| Per-message cost annotation | `  ↳ {in} in / {out} out  ${cost:.4}` rendered dim (LineKind::System) |
| Status bar token format | `{model} \| {state}{scroll}{workflow} \| in:{n} out:{n} \| ${cost:.4}` where `{n}` uses `format_token_count()` |
| Token abbreviation rule | Values ≥ 1000 render as `1.2k`; exact breakpoint is implementation detail |
| Empty state (no messages) | Splash screen — existing splash widget; no change in Phase 1 |
| Error state | `LineKind::Error` renders with `Palette::ERROR` (Red) bold — no copy change in Phase 1 |
| Destructive actions | None in Phase 1 scope |
| Workflow panel border title | Current stage name (dynamic) rendered `Palette::ACCENT` bold |
| Input prompt prefix | `[{model}] > ` rendered `Palette::ACCENT` |

Source: CONTEXT.md D-11/D-12/D-13, user specifics ("↳ dim annotation mirrors opencode pattern")

## Interaction Contract

### Markdown Rendering Gate

- Streaming lines (`is_final: false`): render as plain text — no markdown parsing, no flicker.
- Finalized lines (`is_final: true`): parse with pulldown-cmark 0.12.2, emit ratatui `Span` vec with styled segments.
- Finalization trigger: `\n` receipt in `TextDelta` handler sets `is_final = true` on the completed line; `AgentEvent::AgentEnd` unconditionally finalizes the last open line.
- Scope: `LineKind::Assistant` only. All other line kinds bypass parser and render as plain text.

Source: CONTEXT.md D-07/D-08/D-09/D-10, RESEARCH.md MD-04 row

### AppLayout Struct Contract

Two layout variants must be named, not indexed:

| Field | Variant: wf_active = true | Variant: wf_active = false |
|-------|--------------------------|---------------------------|
| `output` | `Rect` (Min 3) | `Rect` (Min 3) |
| `workflow` | `Some(Rect)` (Length 4) | `None` |
| `status` | `Rect` (Length 1) | `Rect` (Length 1) |
| `input` | `Rect` (Length 3) | `Rect` (Length 3) |

All `chunks[N]` references in `app.rs` are replaced with named field access. No raw index offsets remain after Phase 1.

Source: CONTEXT.md D-05/D-06, RESEARCH.md Pattern 2

### Per-Message Annotation Trigger

- Emitted on `AgentEvent::AgentEnd` (carries `Usage` payload).
- Not emitted during streaming.
- Appended as a new `OutputLine { kind: LineKind::System, is_final: true }` to the output vec.

Source: CONTEXT.md D-12/D-13, RESEARCH.md TOK-01 row
