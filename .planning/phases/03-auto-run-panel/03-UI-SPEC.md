## Spacing Scale

Ratatui uses terminal cell units (columns √ó rows), not CSS pixels. The
equivalent 8-point scale maps as follows. All constraints must be declared
in cell units only ‚Äî no fractional values.

| Token | Cell value | Usage                                        |
|-------|-----------|----------------------------------------------|
| xs    | 1 cell    | Icon-to-label gap (`"  "` separator between pipeline stages) |
| sm    | 2 cells   | Inline padding between spans on a single row |
| md    | 4 cells   | Label prefix before progress bar             |
| lg    | 8 cells   | Right-side count suffix (`  N/M tasks`)      |
| border| 1 cell each side | Block border (ALL borders, existing pattern) |

Exceptions:
- Panel title: 1-space padding inside title string on each side (` {text} `) ‚Äî existing pattern from `render_workflow_panel`.
- Auto-mode panel height: output area shrinks by **7 rows** to accommodate 5 new inner rows plus existing 2 rows plus 2 border rows (per D-01, D-02).

Source: CONTEXT.md D-01, D-02; `widgets.rs:424‚Äì478`.

## Typography

Terminal text has no CSS font-size. Roles map to ratatui `Style` modifiers.

| Role         | Ratatui equivalent         | Usage                                        |
|--------------|---------------------------|----------------------------------------------|
| Body         | `Style::default()`        | Default text ‚Äî no modifier, `Palette::TEXT`  |
| Label        | `.bold()`                 | Blueprint title in panel border title span   |
| Subdued      | `Palette::DIM` (DarkGray) | Borders, footer hints, pipeline pending `‚óã`, empty bar fill |
| Accent/Active| `Palette::ACCENT` (Yellow)| Active stage `‚óè`, context bar fill, current stage name in title |

Weight contract (2 weights only, per typography rules):
- Regular (no modifier): body text, git commit line, stage+section row, model+blueprint row
- Bold (`.bold()`): blueprint ID in panel border title only

Line height: 1 row per content line ‚Äî terminal cells, not CSS line-height.

### Row-by-row render contract

**Row 2 ‚Äî Model + Blueprint (NEW)**
- Left segment: `{model_display}` ‚Äî `Style::default().fg(Palette::TEXT)`
- Separator: `  ‚îÇ  ` ‚Äî `Style::default().fg(Palette::DIM)`
- Right segment: `{blueprint_id}` ‚Äî `Style::default().fg(Palette::TEXT)`
- Truncation: if combined width exceeds panel inner width, truncate `blueprint_id` to `panel_width - model_width - 5` chars with trailing `‚Ä¶`
- Column split point: left segment takes min(model string length + 2, panel_width / 2) columns

**Row 3 ‚Äî Stage + Section (NEW)**
- Format: `{current_stage} ‚Ä∫ {section_or_task}`
- Stage name: `Style::default().fg(Palette::ACCENT)`
- ` ‚Ä∫ `: `Style::default().fg(Palette::DIM)`
- Section/task name: `Style::default().fg(Palette::TEXT)`
- Truncation: trim section/task to 30 chars with `‚Ä¶` if string length exceeds 30

**Row 4 ‚Äî Last Git Commit (NEW)**
- Format: `{short_hash} {commit_subject}`
- Hash: `Style::default().fg(Palette::DIM)` (7 chars, e.g. `abc1234`)
- Single space separator
- Subject: `Style::default().fg(Palette::TEXT)`
- Truncation: trim subject to `panel_inner_width - 9` chars with `‚Ä¶`
- Empty state (before first fetch): render single dim string `‚Äî` centered

**Row 5 ‚Äî Context % Bar (NEW)**
- Format: `ctx {pct}%  {filled}{empty}`
- Label `ctx {pct}%`: `Style::default().fg(Palette::TEXT)` ‚Äî right-pad to 8 chars
- Two-space gap between label and bar
- Filled cells `‚ñà`: `Style::default().fg(Palette::ACCENT)`
- Empty cells `‚ñë`: `Style::default().fg(Palette::DIM)`
- Bar width: `panel_inner_width - 10` cells
- Filled count: `(pct as f64 / 100.0 * bar_width as f64) as u16`
- Zero state (`context_pct == 0`): render full empty bar with label `ctx  ?%`

**Row 6 ‚Äî Footer Hints (NEW)**
- Format: `esc pause  ‚îÇ  ? help`
- All text: `Style::default().fg(Palette::DIM)` (matches Phase 2 footer hint pattern)
- No bold, no accent ‚Äî purely informational

## Interaction Contract

### Auto-Mode Activation / Deactivation
- Panel expands to 7-row layout on `AgentEvent::AutoModeStart` ‚Äî immediate next render tick
- Panel collapses to 2-row layout on `AgentEvent::AutoModeEnd` ‚Äî immediate next render tick (D-13: reset `is_auto` immediately, do not wait for render tick)
- No animation or transition ‚Äî cell replacement is instantaneous

### Git Commit Refresh
- Triggered on `WorkflowUnitEnd` event (D-05)
- Async fetch; render shows last known value until next update
- No spinner or loading indicator ‚Äî empty state is `‚Äî`

### Context % Update
- Updated on every `AgentEvent::ContextUsage { pct: u8 }` event (D-08)
- Bar redraws each frame using current `context_pct` value

### Keybind Hints (Row 6)
- `esc` triggers pause (existing cancel flow) ‚Äî no new binding added in Phase 3
- `?` triggers help overlay (Phase 2) ‚Äî no new binding added in Phase 3
- Footer is display-only, not interactive

Source: CONTEXT.md D-11, D-12, D-13, D-05.

## Copywriting Contract

| Element               | Copy                                                        |
|-----------------------|-------------------------------------------------------------|
| Model+blueprint row   | `{model_display}  ‚îÇ  {blueprint_id}` ‚Äî no label prefix    |
| Stage+section row     | `{Stage} ‚Ä∫ {section-or-task}` ‚Äî Stage is capitalized, section lowercase |

| Footer hints          | `esc pause  ‚îÇ  ? help` ‚Äî lowercase, two spaces around pipe |
| Git empty state       | `‚Äî` (em dash, single char, dim style)                      |
| Panel border title    | ` {blueprint_id} ` (left) + ` {CurrentStage} ` (right, accent) ‚Äî existing pattern |

Primary CTA: none ‚Äî Phase 3 adds display-only rows; no user-initiated actions.

Destructive actions: none in Phase 3.

Error state: no error display within the workflow panel itself. Errors surface via toast system (Phase 2 / `Palette::ERROR`).
