## Spacing Scale

Ratatui uses terminal cell units (columns × rows), not CSS pixels. The
equivalent 8-point scale maps as follows. All constraints must be declared
in cell units only — no fractional values.

| Token | Cell value | Usage                                        |
|-------|-----------|----------------------------------------------|
| xs    | 1 cell    | Icon-to-label gap (`"  "` separator between pipeline stages) |
| sm    | 2 cells   | Inline padding between spans on a single row |
| md    | 4 cells   | Label prefix before progress bar             |
| lg    | 8 cells   | Right-side count suffix (`  N/M tasks`)      |
| border| 1 cell each side | Block border (ALL borders, existing pattern) |

Exceptions:
- Panel title: 1-space padding inside title string on each side (` {text} `) — existing pattern from `render_workflow_panel`.
- Auto-mode panel height: output area shrinks by **7 rows** to accommodate 5 new inner rows plus existing 2 rows plus 2 border rows (per D-01, D-02).

Source: CONTEXT.md D-01, D-02; `widgets.rs:424–478`.

## Typography

Terminal text has no CSS font-size. Roles map to ratatui `Style` modifiers.

| Role         | Ratatui equivalent         | Usage                                        |
|--------------|---------------------------|----------------------------------------------|
| Body         | `Style::default()`        | Default text — no modifier, `Palette::TEXT`  |
| Label        | `.bold()`                 | Blueprint title in panel border title span   |
| Subdued      | `Palette::DIM` (DarkGray) | Borders, footer hints, pipeline pending `○`, empty bar fill |
| Accent/Active| `Palette::ACCENT` (Yellow)| Active stage `●`, context bar fill, current stage name in title |

Weight contract (2 weights only, per typography rules):
- Regular (no modifier): body text, git commit line, stage+section row, model+blueprint row
- Bold (`.bold()`): blueprint ID in panel border title only

Line height: 1 row per content line — terminal cells, not CSS line-height.

Source: `widgets.rs:11–21`, `widgets.rs:408–415`, CONTEXT.md D-15, D-16.

## Color

All colors use ANSI named values only — `Color::Rgb` is forbidden (per Phase 1 D-04 / STATE.md).

| Role            | Ratatui value       | Usage                                                                 |
|-----------------|---------------------|-----------------------------------------------------------------------|
| Dominant (60%)  | Terminal default bg | Output area, panel inner background — no explicit bg set             |
| Secondary (30%) | `Palette::DIM` (Color::DarkGray) | Borders, panel title border style, pending pipeline markers, empty bar cells |
| Accent (10%)    | `Palette::ACCENT` (Color::Yellow) | See reserved list below                                  |
| Success         | `Palette::SUCCESS` (Color::Green) | Pipeline done `✓` markers, existing progress bar fill   |
| Error           | `Palette::ERROR` (Color::Red)     | No new destructive elements in Phase 3                  |
| Status bg       | `Palette::STATUS_BG` (Color::DarkGray) | Status bar only — not used in workflow panel       |

**Accent reserved for (explicit list):**
1. Active pipeline stage `●` marker
2. Context `%` mini-bar fill (`█` filled cells in ctx bar row)
3. Current stage name span in panel border title (` Build `)
4. No other elements — do not apply Yellow to body text, git rows, or model rows

Source: `widgets.rs:11–21`, CONTEXT.md D-15, D-16, Phase 1 D-04.

## Panel Layout Contract

The auto-mode panel is an **extension of `render_workflow_panel()`**. The
inner layout changes from 2 rows to 7 rows when `workflow_state.is_auto == true`
AND `workflow_state.active == true`.

```
┌─ blueprint-name ─────────────────────────┐
│ Define ✓  Explore ✓  Build ●  Validate ○  │  row 0: pipeline (UNCHANGED)
│ ████████░░░░  6/12 tasks                  │  row 1: progress bar (UNCHANGED)
│ claude-opus-4-6  │  blueprint-name        │  row 2: NEW model + blueprint id
│ Build › section-03                        │  row 3: NEW stage › section/task
│ abc1234 feat: scaffold section-03         │  row 4: NEW last git commit
│ ctx 34%  ██████░░░░░░░░░░░░               │  row 5: NEW context % bar
│ esc pause  │  ? help                      │  row 6: NEW footer hints
└──────────────────────────────────────────┘
```

Source: CONTEXT.md `<specifics>` confirmed mockup.

### Row-by-row render contract

**Row 2 — Model + Blueprint (NEW)**
- Left segment: `{model_display}` — `Style::default().fg(Palette::TEXT)`
- Separator: `  │  ` — `Style::default().fg(Palette::DIM)`
- Right segment: `{blueprint_id}` — `Style::default().fg(Palette::TEXT)`
- Truncation: if combined width exceeds panel inner width, truncate `blueprint_id` to `panel_width - model_width - 5` chars with trailing `…`
- Column split point: left segment takes min(model string length + 2, panel_width / 2) columns

**Row 3 — Stage + Section (NEW)**
- Format: `{current_stage} › {section_or_task}`
- Stage name: `Style::default().fg(Palette::ACCENT)`
- ` › `: `Style::default().fg(Palette::DIM)`
- Section/task name: `Style::default().fg(Palette::TEXT)`
- Truncation: trim section/task to 30 chars with `…` if string length exceeds 30

**Row 4 — Last Git Commit (NEW)**
- Format: `{short_hash} {commit_subject}`
- Hash: `Style::default().fg(Palette::DIM)` (7 chars, e.g. `abc1234`)
- Single space separator
- Subject: `Style::default().fg(Palette::TEXT)`
- Truncation: trim subject to `panel_inner_width - 9` chars with `…`
- Empty state (before first fetch): render single dim string `—` centered

**Row 5 — Context % Bar (NEW)**
- Format: `ctx {pct}%  {filled}{empty}`
- Label `ctx {pct}%`: `Style::default().fg(Palette::TEXT)` — right-pad to 8 chars
- Two-space gap between label and bar
- Filled cells `█`: `Style::default().fg(Palette::ACCENT)`
- Empty cells `░`: `Style::default().fg(Palette::DIM)`
- Bar width: `panel_inner_width - 10` cells
- Filled count: `(pct as f64 / 100.0 * bar_width as f64) as u16`
- Zero state (`context_pct == 0`): render full empty bar with label `ctx  ?%`

**Row 6 — Footer Hints (NEW)**
- Format: `esc pause  │  ? help`
- All text: `Style::default().fg(Palette::DIM)` (matches Phase 2 footer hint pattern)
- No bold, no accent — purely informational

Source: CONTEXT.md D-15, D-16; `widgets.rs:447–478` (bar pattern to copy).

## Interaction Contract

### Auto-Mode Activation / Deactivation
- Panel expands to 7-row layout on `AgentEvent::AutoModeStart` — immediate next render tick
- Panel collapses to 2-row layout on `AgentEvent::AutoModeEnd` — immediate next render tick (D-13: reset `is_auto` immediately, do not wait for render tick)
- No animation or transition — cell replacement is instantaneous

### Git Commit Refresh
- Triggered on `WorkflowUnitEnd` event (D-05)
- Async fetch; render shows last known value until next update
- No spinner or loading indicator — empty state is `—`

### Context % Update
- Updated on every `AgentEvent::ContextUsage { pct: u8 }` event (D-08)
- Bar redraws each frame using current `context_pct` value

### Keybind Hints (Row 6)
- `esc` triggers pause (existing cancel flow) — no new binding added in Phase 3
- `?` triggers help overlay (Phase 2) — no new binding added in Phase 3
- Footer is display-only, not interactive

Source: CONTEXT.md D-11, D-12, D-13, D-05.

## Copywriting Contract

| Element               | Copy                                                        |
|-----------------------|-------------------------------------------------------------|
| Model+blueprint row   | `{model_display}  │  {blueprint_id}` — no label prefix    |
| Stage+section row     | `{Stage} › {section-or-task}` — Stage is capitalized, section lowercase |
| Git commit row        | `{abc1234} {commit subject}` — no label prefix             |
| Context bar label     | `ctx {N}%` — lowercase, no colon                           |
| Context bar unknown   | `ctx  ?%` — two spaces before `?` to maintain alignment    |
| Footer hints          | `esc pause  │  ? help` — lowercase, two spaces around pipe |
| Git empty state       | `—` (em dash, single char, dim style)                      |
| Panel border title    | ` {blueprint_id} ` (left) + ` {CurrentStage} ` (right, accent) — existing pattern |

Primary CTA: none — Phase 3 adds display-only rows; no user-initiated actions.

Destructive actions: none in Phase 3.

Error state: no error display within the workflow panel itself. Errors surface via toast system (Phase 2 / `Palette::ERROR`).

Source: CONTEXT.md D-15, D-16, `<specifics>` mockup.

## Truncation Rules

| Field            | Max chars | Truncation char |
|------------------|-----------|-----------------|
| blueprint_id     | 30        | `…`             |
| model_display    | panel_width / 2 - 5 | `…`   |
| commit subject   | panel_inner_width - 9 | `…` |
| section/task name| 30        | `…`             |

All truncation appends `…` (U+2026 HORIZONTAL ELLIPSIS) — single char, not `...`.
